Sites

French For all nouveau

Un site Web multi-pages entièrement construit à l'aide de la technologie de pointe de Next.js 13. Ce site comprend des images entièrement optimisées, une conception d'interface utilisateur moderne, une page personnalisée 404 introuvable et une intégration calendly. Le site a été soigneusement optimisé pour le référencement afin de maximiser la visibilité organique avec tous les mots-clés appropriés ajoutés dans les balises méta avec une méta description simple qui s'intègre dans l'espace de pixels limité de Google. Le client voulait faire passer sa petite entreprise de tutorat à l'étape suivante, et une partie importante de la réalisation de cette était de créer un site Web professionnel et à chargement rapide qui attirerait plus de clients et les canaliserait dans un système de planification qui optimiserait le processus d'intégration des étudiants.

KX KDA

Un site Web en cours de développement qui suivra bientôt toutes les données pertinentes sur la crypto-monnaie "Kadena" (KDA). Avec ce projet parallèle, je voulais utiliser certaines de mes connaissances approfondies en crypto-monnaie et NFT et me mettre au défi de construire un application utile qui peut être utilisée pour suivre cet écosystème émergent. Cette application vise à suivre non seulement tous les jetons qui sont actuellement activement échangés sur la blockchain Kadena, mais cherchera également des informations pertinentes sur les projets qui doivent être lancés sur la chaîne. Il s'agit de l'application précurseur d'une application de crypto-monnaie plus vaste qui est configurée pour suivre des données beaucoup plus complexes.

HRM

Un site Web de blog JAM stack créé pour une mission universitaire complet avec un backend CMS "headless" Sanity et un frontal Next.js en utilisant la régénération statique incrémentielle pour améliorer la vitesse de chargement et de mise à jour des pages. Des mises à jour rapides peuvent être apportées aux articles de blog à partir du Sanity Studio intégré, accessible uniquement par parties autorisées en tapant /studio à la fin de l'URL du site Web. Cela signifie que le backend fonctionne sur la même plate-forme d'hébergement que le site Web du blog, ce qui permet de réaliser des économies importantes.
